# proposals/

Populated in Phase 4 (see [METHODOLOGY.md](../METHODOLOGY.md)). Each
`NNN-<slug>.md` turns a finding from `analysis/findings.md` into a concrete
proposal shaped like a [harness-score check-change issue](https://github.com/paladini/harness-score/blob/main/CONTRIBUTING.md#adding-or-changing-a-check) —
naming the check, dimension, points, and what fixture/doc changes it
implies. Filed as issues against
[harness-score](https://github.com/paladini/harness-score), not implemented
in this repository.
